► What is Cubescript?

Cubescript is a compiled scripting language built for SpongeForge mod workflows. You write .cube files that describe game logic — event handlers, world queries, item data — and the compiler catches scope errors and bad references at compile time, before the code ever touches a running Minecraft server.

Cubescript output is JavaScript loaded by a companion Sponge plugin via GraalVM's Polyglot API, running directly inside the JVM — no external Node process, no separate runtime. Mods and Sponge plugins co-exist on the same server through SpongeForge, and Cubescript slots cleanly into that pipeline.

Declarations use the keyword place (not let). Under the hood: Ohm parser → typed AST → static analyzer → constant-folding optimizer → JavaScript codegen. The full grammar is in src/cubescript.ohm.

.cube source → Ohm parse → AST → Analyze → Optimize → JS codegen → GraalVM / Sponge

12 · Mod recipe (nested calls) 12-mod-recipe.cube ↗

Cubescript
Generated JavaScript
mine craft_planks(log_count) {
  return log_count * 4;
}

mine craft_sticks(plank_count) {
  return plank_count / 2;
}

mine can_craft_pickaxe(logs) {
  place planks = craft_planks(logs);
  place sticks = craft_sticks(planks);
  return planks >= 3 && sticks >= 2;
}

place enough = can_craft_pickaxe(2);
enough;
function craft_planks(log_count) {
  return (log_count * 4);
}
function craft_sticks(plank_count) {
  return (plank_count / 2);
}
function can_craft_pickaxe(logs) {
  let planks = craft_planks(logs);
  let sticks = craft_sticks(planks);
  return ((planks >= 3) && (sticks >= 2));
}
let enough = can_craft_pickaxe(2);
enough;
